# Contributing to Chess Advisor

Thanks for your interest in contributing! This project is intentionally simple — a single HTML file with no build step — and we'd like to keep it that way.

## How to Contribute

1. **Fork** this repository
2. **Create a branch** for your feature: `git checkout -b feature/my-feature`
3. **Make your changes** to `index.html`
4. **Test** by opening `index.html` in your browser (or use a local server for WASM features)
5. **Submit a Pull Request** with a clear description of what you changed and why

## Guidelines

### Keep It Simple
- The app is a single HTML file. Please don't introduce build tools, bundlers, or frameworks unless absolutely necessary.
- External libraries should be loaded via CDN, not bundled.

### Code Style
- Use consistent indentation (2 spaces)
- Comment complex logic
- Keep functions focused and reasonably sized

### What We're Looking For
- Bug fixes
- Performance improvements
- New analysis features
- UI/UX improvements
- Better mobile support
- Accessibility improvements
- Documentation improvements

### What to Avoid
- Breaking the single-file architecture without discussion
- Adding heavy dependencies
- Features that require a backend server
- Copyrighted assets (piece sets, board themes from commercial sites)

## Reporting Bugs

Open an issue with:
- Browser and version
- Steps to reproduce
- Expected vs actual behavior
- Console errors (if any)

## Feature Requests

Open an issue with the `enhancement` label. Describe:
- What problem it solves
- How you envision it working
- Whether you'd be willing to implement it

## Questions?

Open a discussion or issue — happy to help!
